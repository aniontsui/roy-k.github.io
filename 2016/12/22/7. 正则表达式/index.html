正则表达式

正则基础

  1. . 任意字符（除换行符以外）

  2. \d 数字0-9

  3. \D 非\d，即不是数字0-9的字符

  4. \w 数字0-9 或字母a-z及A-Z 或下划线

  5. \W 非\w

  6. \s 空格符、tab、换页符、换行符

  7. \S 非\s

  8. \t \r \n \v \f tab 回车 垂直制表符 换页符

范围符号

  1. […] 字符范围

  2. [^..] 字符范围之外

  3. ^ 行首

  4. $ 行尾

  5. \b 零宽单词边界

  6. \B 非\b

特殊字符转义

\ /\^abc/.test(‘^abc’);

分组

  1. (x) 分组，并记录匹配到的字符串

     (abc){3}  abc重复3次

  2. \n 表示使用分组符(x)匹配到字符串

  3. (?:x) 仅分组，即不记录 /(?:abc)(def)\1/

  4. $1 反向引用

‘2015-12-25’.replace(/(\d{4}-(\d{2}-(\d{2})/g,’$2/$3/$1’)

2015-12-25 至 12/25/2015

前瞻

  1. 前瞻

    • exp(?=assert) 正向前瞻 后面等于条件

    • exp(?!assert) 负向前瞻 后面不等于条件

\w(?=\d)

js不支持后顾！

重复

  1. x* x+ 重复次数>=0 重复次数>0 贪婪算法（尽可能匹配到最多）

  2. x*? x+? 非贪婪算法（尽可能匹配少）

  3. x? 出现0或1次

  4. x|y x或y

     ab(cd|ef)g    相对于abcdg 或 abefg

  5. x{n} x{n,}? x{n,m} 重复n次，重复>n次（尽可能少），n< 重复次数 <m

三个Flag

  1. global 全局匹配

  2. ignoreCase 忽略大小写

  3. multiline 多行匹配

正则对象属性

  1. global /abc/g.global = true;

  2. ignoreCase

  3. multiline

  4. source /abc/g.source = ‘abc’;

  5. lastIndex 下一次exec匹配开始的索引

正则对象方法

  1. compile 修改正则（source

  2. exec 获得匹配到的结果 没有成功返回null

  3. test 检测是否有满足正则 有则返回true

  4. toString 把正则表达式转为字符串

string类型与正则相关的方法

  1. String.prototype.search 查找是否有满足的

  2. String.prototype.replace 替换

特殊标记符$

$i : 表示从左到右正则子表达式所匹配的文本。(根据分组）

$& : 表示与正则表达式匹配的全文本。（每次的匹配结果，而不是全部，也不是整个字符串）

$` : 表示匹配字符串的左边文本。（每次匹配结果的左边全部，根据原字符串，而不是替换后的）

$’ : 表示匹配字符串的右边文本。

$$ : 表示$转义。（即替换为$）（如果只有一个，也可写作”$”）

函数

每次匹配成功都会传参数给函数并执行（类似循环）

参数

  1. 整个正则表达式匹配的字符。
  2. 第一分组匹配的内容、第二分组匹配的内容…… 以此类推直到最后一个分组。
  3. 匹配结果在源自符串中的下标（位置）。
  4. 源自符串

  1. String.prototype.match 返回匹配结果 没有成功返回null

  2. String.prototype.split 使用匹配结果分隔字符串，生成数组
