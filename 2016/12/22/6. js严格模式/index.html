2016年 12月 22日

  1. 1. 严格模式

严格模式

严格模式是一种特殊的执行模式，它修复了部分语言上的不足，提供更强的错误检查，并增强安全性。
  1. 不允许用with
  2. 不允许未声明的变量赋值

  3. arguments变为参数的静态副本 传入对象是可以修改其value

    !function(a) {

     'use strict';
 arguments[0].x = 100;
 console.log(a.x);

    }({x:1});

  4. delete参数、函数名报错

    !function(a) {

     'use strict';
 delete a;

    }(1);

  5. delete不可配置的属性报错

    !function(a) {

     'use strict';
 var obj = {};
 Object.defineProperty(obj, 
     'a', {configurable : false});
 delete obj.a;

    }(1);

  6. 对象字面量重复属性名报错

    !function() {

     'use strict';
 var obj = {x : 1, x : 2};

    }();

  7. 禁止八进制字面量

    !function() {

     'use strict';
 console.log(0123);

    }();

  8. eval, arguments变为关键字，不能作为变量、函数名

    !function() {

     'use strict';
 function eval(){}

    }();

  9. eval独立作用域

    !function() {

     'use strict';
 eval('var evalVal = 2;');
 console.log(typeof evalVal);

    }(); // undefined

  10. 一般函数调用时(不是对象的方法调用，也不使用apply/call/bind等修改this)this指向null，而不是全局对象。

若使用apply/call，当传入null或undefined时，this将指向null或undefined，而不是全局对象。
