原型与继承

面向对象

面向对象是对面向过程的一个封装

优势

封装代码，使代码更好维护

减少全局污染

将功能相近的代码组织到一起维护方便，找错方便

实现方式

  1. 将数据与功能封装成函数(方法)

  2. 将相关的功能绑定到一起(对象)

  3. 将功能进行分组(模块)

原型

1. 对象的原型是什么 构造函数的prototype属性的值

2. 如何获得原型对象: 构造函数的prototype属性 和 实例的__proto__

结论:

  1. 任何函数都可以作为构造函数，函数默认就有一个prototype 属性

  2. 由构造函数创建出来的对象会默认链接到其构造函数的prototype属性上 – 对象可以使用其构造函数prototype属性中的成员.

  3. 函数的prototype属性的类型是: object

  4. 原型的作用: 数据共享–继承

  5. 新创建的空对象继承自原型对象

  6. 对象在被创建的那一刻, 原型已经定下来了(new 设置),

    p.proto 与 Person.prototype 的区别

    • 唯一的区别: 从不同角度来访问 原型对象 (实例/构造函数)
    • F.prototype 原型属性 – 构造函数F的
    • f.proto 原型对象 – 对象f的

原型链中的三角关系

实例.__proto__ === 原型对象
构造函数.prototype === 原型对象
原型对象.constructor === 构造函数

注意: proto是一个非标准的属性, 就是不能用于实际的项目开发中 – 用于调试

new的内部原理

1. 创建一个新的空对象
2. 将构造函数的作用域赋给新对象（所以this就指向了这个新对象）
3. 把空对象的内部原型指向构造函数的原型对象
4. 当构造函数执行完成后,如果没有return的话,那么把当前空对象返回
5. **其实new操作主要是把原型链跟实例的this关联起来**

函数 / 构造函数 / 对象 / 实例

函数: 代码复用

构造函数: 创建对象代码的复用

实例: 由构造函数创建出来的对象

对象: 泛指javascript中的复杂数据类型

对象成员: 

属性: 属性值是一个数据
方法: 属性值是一个函数--如果函数挂载在一个对象上,作为对象的一个属性, 就称它为对象的方法

封装

函数 -- 对 具体过程 的封装
对象 -- 对 行为/数据 的封装

javascript 继承

子类可以使用从父类继承的属性和方法

（原型链角度）
每一个对象都有一条原型链存在，所谓的原型继承就是同个任何的手段来改变原型链的层次结构，
使对象访问到原型链中的所有原型对象的属性和方法，从而实现了继承

属性修改和读取的区别

  • 获取操作：

    遵循属性查找原则：即沿着原型链一层层往上找。

  • 赋值操作（只与当前对象有关）：

    只会操作对象本身，如果对象中没有该属性则创建该属性，并赋值；如果对象中有，则修改

  1. 原型继承

    如果需要让一个对象拥有某些属性和方法，可以把这些属性和方法放到原型对象中，因为对象会继承自原型对象，所以就获得了该属性和方法。

    1. P.prototype.do = fn; // 利用对象的动态性, 给原型添加成员
    2. Person.prototype = {f1:fn,f2:fn, // 利用覆盖原型对象的方式添加成员
       constructor:Person;        // 如果用到了, 需要手动添加
      }
    3. 利用混入式继承, 给原型添加extend方法来扩展: Person.prototype.extend=fn

  2. 混入式继承

    一个对象是可以继承自多个对象的,相对于把其他对象的属性放到了 该对象中(遍历赋值)

    for(var k in o1) {

     obj[k] = o1[k];

    }

    obj.extend = function(o) { // 封装继承的方式

     for(var k in o) {
     this[k] = o[k];
 }

    }

  3. 经典继承

    p1 = Object.create(p);

  4. 寄生式继承

    var p = new Person(‘’); Person(a){ a return {a:a} }

  5. 寄生组合式继承

  6. 借用构造函数等

  7. 工厂模式

    var p1 = Person(‘’); Person(a){ a return {a:a} }

静态成员与实例成员：

构造函数上的成员，可以通过构造函数直接访问，是静态成员
